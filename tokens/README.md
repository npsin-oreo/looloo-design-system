# tokens/ тАФ canonical token layer (DS v2)

Status: **Phase 1 тАФ structure only.** Nothing in the app, Storybook, or the
published package consumes these files yet. The legacy pipeline
(`tokens.json` тЖТ `app/primitives.css`, `brand.config.json` тЖТ `app/brand.css`)
remains 100% authoritative until Phase 2 generates and verifies
`dist/tokens/*.css`.

## Layers

| Dir | Layer | May reference | Consumed by |
|---|---|---|---|
| `raw/` | legacy exports (byte copies) | тАФ | migration only тАФ never components |
| `primitive/` | raw values, no UI meaning (`color.blue.500`, `space.2`) | nothing | semantic/component tokens |
| `semantic/` | UI meaning (`color.primary`, `status.success`) | primitive | components, themes |
| `component/` | per-component decisions (`button.height.md`) тАФ **Phase 4** | semantic + primitive | component variants |
| `mode/` | light/dark/density/contrast overrides | existing token paths | build pipeline |
| `theme/` | brand personality (radius, fonts, accents) | semantic + component paths | brand builds |

## Generated vs hand-owned

```bash
npm run tokens:migrate   # regenerates the GENERATED set below from tokens.json + brand.config.json
```

| Generated by `tokens:migrate` тАФ **never hand-edit** | Hand-owned тАФ edit directly |
|---|---|
| `raw/*` (byte copies) | `semantic/{surface,content,border,interaction,focus}.json` |
| `primitive/*.json` (all 10) | `component/*.json` (17 files, captured from component source) |
| `semantic/{color,status}.json` | `mode/{light,compact,comfortable,high-contrast}.json` |
| `mode/dark.json` | `theme/{looloo,healthcare,dashboard}.json` |
| `theme/neutral.json` | |

Hand-owned files reference generated paths via `{alias}` тАФ the Phase 2 validator
checks that those paths exist. Tokens/modes/themes that nothing consumes yet are
marked `$extensions["looloo.status"] = "proposed"` or `PROPOSED` in the file
`$description`. See `docs/tokens/token-source-strategy.md` for the full policy.

Naming note: DTCG token names must not contain `.`, so fractional spacing steps
are hyphenated тАФ Tailwind `px-2.5` тЖФ `{space.2-5}`.

## Format

Practical DTCG: `$type`, `$value`, `$description`, `$deprecated`, `$extensions`.
Aliases use `{dot.path}`. `$extensions` namespaces used:

- `looloo.source` тАФ where the value came from in the legacy export
- `looloo.legacy` тАФ original raw value before normalization (px, hex, `{ref}`)
- `looloo.resolved` тАФ resolved OKLCH for an aliased semantic token
- `looloo.status: "proposed"` тАФ new v2 token not consumed by any component yet

## Phase 1 decisions

1. **Radix palette not promoted.** `rdx-*` has zero consumers in `app/` and
   `components/` тАФ it stays in `raw/legacy.tokens.json` for reference only.
2. **Source prefixes dropped.** `tw-blue-500` тЖТ `color.blue.500`;
   brand ramps тЖТ `color.brand.primary.500` (the `brand.` segment avoids
   colliding with the *semantic* `color.primary`).
3. **Flat kebab semantic keys.** `primary-foreground` (not
   `primary.foreground`) so token names map 1:1 to the existing CSS custom
   properties and to `token-contract.json`.
4. **px тЖТ rem** (├╖16) for spacing/radius/font sizes; original value kept in
   `$extensions["looloo.legacy"]`.
5. **`brand.config.json` remains the brand-override input.** Semantic files
   are *generated from it* so brand branches keep working unchanged; run
   `tokens:migrate` after a brand edit.

## Rules (enforced by validation from Phase 2)

- Components never import from `raw/`.
- Semantic tokens must not reference component tokens.
- Component tokens must not reference raw tokens.
- No source names (`tw`, `rdx`, `shadcn`) in canonical token paths.
- Deprecations need a replacement note before deletion.
